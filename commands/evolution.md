# /as-me:evolution

查看原则的演化历史和时间线，追踪记忆如何随时间变化。

## 演化触发类型

- **new_evidence** (新证据): 新的记忆支持现有原则
- **conflict** (冲突证据): 发现与现有原则矛盾的证据
- **user_confirm** (用户确认): 用户确认原则正确
- **user_correct** (用户修正): 用户修正原则内容
- **time_decay** (时间衰减): 长时间未被触发导致置信度下降
- **aggregation** (记忆聚合): 从多个记忆聚合形成新原则

## 使用方法

### 查看演化历史

查看特定原则的完整演化历史：

```
/as-me:evolution history -p <PRINCIPLE_ID>
```

查看所有演化事件：

```
/as-me:evolution history
```

选项：
- `--principle, -p <ID>`: 指定原则 ID
- `--limit, -n <NUM>`: 显示数量限制（默认 20）
- `--verbose, -v`: 显示详细信息

### 查看演化时间线

按时间线查看所有演化事件：

```
/as-me:evolution timeline
```

选项：
- `--trigger, -t <TYPE>`: 按触发类型过滤
- `--limit, -n <NUM>`: 显示数量限制（默认 30）
- `--verbose, -v`: 显示详细信息

### 查看事件详情

```
/as-me:evolution show <EVENT_ID>
```

## 示例

查看特定原则的演化历史：
```
/as-me:evolution history -p abc123ef
```

查看所有用户确认事件：
```
/as-me:evolution timeline -t user_confirm
```

查看详细的时间线：
```
/as-me:evolution timeline -v
```

## 输出格式

时间线格式：
```
共 5 条演化事件

[01-16 10:00] [abc123ef] [用户确认] ↑ 80% → 100%
[01-15 15:30] [def456ab] [新证据] ↑ 60% → 70%
[01-14 09:00] [abc123ef] [记忆聚合] = 80%
```

原则演化历史格式：
```
原则 abc123ef 的演化时间线

时间线:
------------------------------------------------------------
  2026-01-14 09:00:00
  │ 记忆聚合: 0% → 80%
  │ 从 5 条技术偏好记忆聚合形成
  │
  2026-01-15 15:30:00
  │ 新证据: 80% → 85%
  │ 发现新的支持证据
  │
  2026-01-16 10:00:00
  │ 用户确认: 85% → 100%
  │ 用户确认原则
  │
------------------------------------------------------------
```

## 演化机制

原则的演化遵循以下规则：

1. **新证据**: 每条新的支持证据提升 1-5% 置信度
2. **冲突证据**: 可能降低置信度或触发原则分裂
3. **用户确认**: 提升 20% 置信度（上限 100%）
4. **用户修正**: 原则内容更新，置信度可能调整
5. **时间衰减**: 长时间未触发的原则置信度逐渐降低
6. **记忆聚合**: 相似记忆聚合时创建新原则

通过演化追踪，您可以了解系统对您的理解是如何随时间发展和完善的。
